# Contributing

Keep this repo practical.

Good contributions:

- Concrete production risks
- Reusable launch gates
- Agent prompts with clear output formats
- Lightweight checks with low false-positive rates
- Templates teams can actually fill out

Avoid:

- Generic startup advice
- Unsourced claims
- Vendor hype
- Heavy frameworks
- Compliance promises

When adding a checklist item, prefer this shape:

```text
- [ ] Specific condition that can be inspected or tested.
```
